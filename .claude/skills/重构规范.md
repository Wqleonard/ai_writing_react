渐进式重构

如果识别到指令为类似 将xxx.vue（可能是文件引用/文件夹引用） ，生成/重构为 react 类似的命令，启用本rules
启用本rules 时，在回复中称呼我为 重构之王

目标：
    - 从vue3 切换为react18+
    - 不妨碍现有项目正常迭代
    - 重构完成后会删除所有vue相关依赖，重构中不进行删除

规范：
    - 如果vue3项目中使用到了静态资源，重构后向 @/assests 中复制一份
    - react 组件，优先尝试使用 shadcn/ui  如果遇到没有满足需求的组件时， 先询问我如何处理，是否自己实现
    - 样式优先采用 tailwind 实现，如果遇到实在无法实现的，先询问我，如何处理
    - pages 下的页面文件，采取kebab-case（短横线）连接，如 creation-community
    - components 下的组件文件，采取PascalCase（大驼峰）如 MarkdownEditor
    - router 下的路由路径，采取kebab-case（短横线）连接，如 /creation-community/user
    - 优先使用箭头函数来声明组件，非必要不使用function来进行声明
    - 使用  shadcn/ui 下载的组件源码，放置于  /react-app/src/components/ui   下
    - 优先使用 @/components/ui 下的ui组件，如果没有对应组件，先查询shadcn/ui 中是否有对应组件，如果有则在 @/components/ui 下新建，如果没有，询问我后实现
    - 使用 ScrollArea 替换 el-scrollbar 组件
    - react中 className 如果需要拼接，使用库 cn ；import { cn } from "@react-app/lib/utils";
    - api 类的重构， 优先查找 @react-app/api 下有没有对应的重构后的api，如果没有，重新实现在 @react-app/api 下
    - 埋点相关功能在重构时不需要实现
    - 实现按钮类标签时，优先使用 @/components/ui 下的 Button 组件
    - react端用到的状态管理库为 zustand。 放置与目录 @react-app/store  下
    - 根据最佳实践， store使用zustand, sotre和types应该分文件放置
    - vue 版文件使用Unicode+iconfont实现的图标，重构为react版本后，使用 @/componets/Iconfont 实现
    - 大部分情况下，都不需要使用tabIndex 做Tab 适配。 如果有需要的情况，我会特别提到
    - 封装所有的dialog类，均使用 @reacp-app/components/ui/Dialog 组件。并且至少需要满足 open, onOpenChange 两个props
